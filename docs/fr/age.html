Mon âge ATS

Saisissez votre date de naissance pour obtenir votre âge en Δ, votre prochain Kilo-versaire et votre situation dans les trois grandes Générations (Apprentissage / Construction / Transmission).

Avis de confidentialité

  • Aucune donnée ne quitte votre navigateur. Le calcul se fait entièrement côté client avec le JavaScript de cette page.
  • Pas d'analytics, pas de tracking, pas de cookies. Le site n'embarque aucune balise tierce et ne poste rien à un serveur.
  • La persistance est opt-in. Le champ date de naissance n'est pas stocké automatiquement. Si vous le remplissez puis rechargez la page, la valeur disparaît — aucune écriture dans localStorage sur cette page.
  • Le téléchargement .ics est local. Le fichier est construit en mémoire par votre navigateur et sauvegardé via le dialogue de téléchargement standard. Nous ne le voyons jamais.
  • La CSP interdit l'exfiltration. connect-src est restreint (cf. la balise <meta> CSP en tête de page) — même si une régression future ajoutait un fetch(), le navigateur le bloquerait.

Comment l'âge ATS est calculé

Soit Δnow le nombre de jours écoulés depuis l'époque ATS (1969-07-20T00:00:00Z) et Δself le nombre de jours écoulés depuis l'époque jusqu'à votre instant de naissance (signé : négatif pour les naissances antérieures à l'époque). Votre âge ATS est simplement la durée :

Δdage  =  Δnow  −  Δself

Le résultat est une durée (Δd), pas un instant (Δ). Cf. conventions §4.1 pour la distinction entre Δ (instant) et Δd (durée) et manifeste §11 pour l'algèbre.

La partie entière est le nombre de jours pleins vécus. La partie décimale est la fraction du jour courant, avec la même précision 5 chiffres que la forme canonique ATS (cf. manifeste §4).

Format d'export .ics

Le téléchargement produit un VCALENDAR standard RFC 5545 avec un VEVENT par jalon à venir (5 Kilo-versaires + 10 Hecto-fêtes, cf. conventions §1 pour les définitions). Résumé des champs :

ChampValeurNotes
PRODID-//ATS//Apollonian Time System//ENIdentifie le producteur.
VERSION2.0iCalendar 2.0 conforme RFC 5545.
CALSCALEGREGORIANRequis par RFC 5545 ; les jalons ATS sont datés par leur jour calendaire grégorien pour la compatibilité avec les apps calendrier.
METHODPUBLISHPublication statique, pas une invitation de réunion.
UIDats-{kind}-{days}-{birthMs}@<origine>Déterministe : même date → mêmes UID entre exports (les apps calendrier mettent à jour en place, pas de doublons).
DTSTAMPmaintenant (UTC)L'instant de génération du fichier.
DTSTART / DTENDJournée entière, VALUE=DATEUn jour UTC complet par jalon. Testé Apple Calendar, Google Calendar, Outlook, Thunderbird.
SUMMARYΔ ATS — Kilo-versary (N days) ou Hecto-celebrationS'affiche comme titre de l'événement.
DESCRIPTIONN days since birth. ATS milestone.Corps de texte pour les clients qui l'affichent.

Compatible avec Apple Calendar (macOS / iOS), Google Calendar (import web), Outlook (Windows / Mac / Web) et Thunderbird Lightning. Pour ré-importer après une révision de la spec ATS, régénérez depuis cette page ; le UID déterministe assure la mise à jour en place.
