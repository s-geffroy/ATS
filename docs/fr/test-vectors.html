Vecteurs de conformance

Le contrat qui transforme ATS d'un projet personnel en standard fiable. Chaque implémentation conformante produit une sortie bit-à-bit identique sur ces 104 vecteurs répartis en 9 jeux. Tables filtrables, copie JSON en un clic, fonctionne hors-ligne (Service Worker depuis v0.6).

Que sont les vecteurs de conformance

Un vecteur de conformance est un cas de test déterministe : un instant UTC plus l'encodage ATS qu'une implémentation doit produire pour cette entrée. L'ensemble de vecteurs ci-dessous est la forme exécutable du contrat de conformance de la spec — les exécuter, c'est ainsi qu'une bibliothèque tierce prouve qu'elle implémente ATS correctement. La définition normative vit dans manifeste §16.5 (conformance) et §9.3 (contrat de conversion) ; la politique d'additivité des vecteurs vit dans versioning §4.

  "epoch_utc": "1969-07-20T00:00:00Z",
  "precision": 5,                  // chiffres fractionnaires par défaut (manifeste §4.4)
  "vectors": [
    {
      "label": "Epoch (start of Apollo 11 landing day)",

}

Le champ spec_version est obligatoire sur chaque fichier de vecteurs (versioning §2). Les fichiers arithmétique et bridges utilisent de petites variations de ce schéma — le contrôleur JS ci-dessous s'adapte automatiquement.

Comment les exécuter

Choisissez l'implémentation de référence la plus proche de votre stack et adaptez :

# Python — couvre les 9 jeux bit-à-bit
docker run --rm -v "$PWD:/app" -w /app python:3.13-slim \
  sh -c 'pip install --quiet ".[bridges,tests]" && python -m unittest discover tests'
// JavaScript — couvre le core Terre + arithmétique
docker run --rm -v "$PWD:/app" -w /app node:20-slim \
  node tests/test_vectors.mjs
# 12/12 JS vectors pass

Revendiquer la conformance à ATS v0.7 signifie passer les jeux core (12) et arithmétique (12) bit-à-bit (manifeste §16.5). Les jeux multi-planétaire et bridges sont optionnels et doivent être déclarés explicitement si revendiqués.

Pourquoi exactement ces nombres

  • 12 core — couvre l'époque, l'instant remarquable du touchdown, les quatre ancrages du jour (Bloc 0/5 = minuit/midi UTC), grand Kilo (vérifie non-bornitude), cas limites fractionnaires (frac=99999 limite), et la branche T-.
  • 12 arithmétique — exerce les sept opérations typées Δ/Δd du §11.4, les retenues Kin → Deka → Hecto → Kilo, le flip de direction au franchissement d'époque, et les comparaisons inter-signes.
  • 10 par corps / bridge — couverture minimale requise par multi-planétaire §6.3 : époque, jalons ronds, fractions complexes, T-, une date 2050+, et instants contemporains.

Pour les implémenteurs

  • Les vecteurs sont additifs uniquement. Un vecteur publié n'est jamais modifié après la release (versioning §4.1). Si une erreur est trouvée, le vecteur corrigé est ajouté et le mauvais marqué deprecated: true.
  • Dérive de spec-version. Un consommateur revendiquant la conformance à vX.Y doit rejeter les fichiers déclarant spec_version > vX.Y et doit passer chaque vecteur non déprécié avec spec_version ≤ vX.Y.
  • La compatibilité ascendante (sauter les vecteurs inconnus avec un avertissement) est RECOMMANDÉE mais non REQUISE.
  • Proposer de nouveaux vecteurs suit la procédure RFC légère dans versioning §6 — minimum 14 jours de commentaire public.
